# Неиспользуемые или опциональные файлы

## Файлы, которые можно удалить (если не используете)

### 1. `nuxt.config.local.ts`
- **Статус**: Не используется автоматически
- **Причина**: Nuxt не подхватывает файлы с суффиксом `.local.ts` автоматически
- **Действие**: Можно удалить, если не планируете использовать локальную конфигурацию

### 2. `setup-https.sh`
- **Статус**: Опциональный
- **Причина**: Скрипт для настройки HTTPS на VPS без Docker
- **Действие**: Можно удалить, если используете Docker Compose (там Nginx в контейнере)

### 3. `deploy.sh`
- **Статус**: Опциональный
- **Причина**: Простой скрипт для деплоя без Docker Compose
- **Действие**: Можно удалить, если используете `docker-compose up`

### 4. `ecosystem.config.js`
- **Статус**: Опциональный
- **Причина**: Используется только для PM2 (деплой без Docker)
- **Действие**: Можно удалить, если используете Docker
- **Примечание**: Исправлен путь к скрипту (было `./server/index.mjs`, стало `./.output/server/index.mjs`)

## Файлы, которые используются

- ✅ `Dockerfile` - для сборки Docker образа
- ✅ `docker-compose.yml` - для запуска с Docker Compose
- ✅ `.dockerignore` - для оптимизации сборки
- ✅ `nginx.conf` - конфигурация Nginx (используется в Docker Compose)
- ✅ `.env.example` - пример переменных окружения
- ✅ `DEPLOY.md` - инструкция по деплою

## Рекомендации

Если вы используете **Docker Compose** (рекомендуется):
- Можно удалить: `setup-https.sh`, `deploy.sh`, `ecosystem.config.js`, `nuxt.config.local.ts`

Если вы используете **PM2 на VPS**:
- Оставьте: `ecosystem.config.js`, `setup-https.sh`
- Можно удалить: `docker-compose.yml`, `Dockerfile`, `.dockerignore`

Если вы используете **облачные платформы** (Vercel, Netlify, Railway):
- Можно удалить: все скрипты и конфигурации для VPS (`setup-https.sh`, `deploy.sh`, `ecosystem.config.js`, `nginx.conf`)
