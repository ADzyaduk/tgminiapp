# Какой репозиторий использовать для Amvera?

Amvera Cloud поддерживает **два варианта** работы с Git. Вы можете выбрать любой, который вам удобнее.

## Вариант 1: Использовать свой репозиторий (GitHub/GitLab/Bitbucket) ⭐ Рекомендуется

**Подходит, если:**
- ✅ У вас уже есть репозиторий на GitHub/GitLab/Bitbucket
- ✅ Вы хотите хранить код в одном месте
- ✅ Вы работаете в команде
- ✅ Вы хотите использовать все возможности GitHub (Issues, PR, Actions и т.д.)

**Как это работает:**
1. Ваш код остается в вашем репозитории (GitHub/GitLab/Bitbucket)
2. Amvera подключается к вашему репозиторию через API
3. При каждом `git push` в ваш репозиторий → Amvera автоматически деплоит

**Настройка:**
1. В панели Amvera: **«Настройки» → «Репозиторий»**
2. Выберите **«Подключить внешний репозиторий»**
3. Авторизуйтесь через GitHub/GitLab/Bitbucket
4. Выберите ваш репозиторий и ветку (обычно `main` или `master`)
5. Готово! Теперь просто делайте `git push` в ваш репозиторий

**Преимущества:**
- ✅ Код в одном месте
- ✅ Используете привычный workflow
- ✅ Легко работать в команде
- ✅ Можно использовать GitHub Actions, CI/CD и т.д.

## Вариант 2: Использовать репозиторий Amvera

**Подходит, если:**
- ✅ Вы только начинаете проект
- ✅ Не хотите создавать репозиторий на GitHub
- ✅ Просто хотите быстро задеплоить

**Как это работает:**
1. Amvera создает для вас репозиторий на `git.amvera.ru`
2. Вы добавляете его как remote в ваш локальный проект
3. Делаете `git push` в репозиторий Amvera
4. Amvera автоматически деплоит

**Настройка:**
1. В панели Amvera: **«Репозиторий»**
2. Скопируйте URL репозитория (например: `https://git.amvera.ru/username/project-name`)
3. В локальной папке проекта:

```bash
# Добавьте remote репозиторий Amvera
git remote add amvera https://git.amvera.ru/username/project-name

# Отправьте код
git push amvera master
# или
git push amvera main
```

**Преимущества:**
- ✅ Быстрый старт
- ✅ Не нужно создавать репозиторий на GitHub
- ✅ Все в одном месте (код и деплой)

**Недостатки:**
- ❌ Код хранится в двух местах (локально и на Amvera)
- ❌ Меньше возможностей для работы в команде
- ❌ Нет GitHub Issues, PR и других инструментов

---

## Какой вариант выбрать?

### Рекомендация: **Вариант 1 (свой репозиторий)**

Если у вас уже есть репозиторий на GitHub/GitLab/Bitbucket, используйте его. Это удобнее и правильнее с точки зрения организации работы.

### Когда использовать Вариант 2:

- Если вы только тестируете Amvera
- Если проект небольшой и не планируете работать в команде
- Если не хотите создавать репозиторий на GitHub

---

## Можно ли использовать оба?

**Технически да**, но это не рекомендуется:
- Код будет в двух местах
- Нужно делать push в оба репозитория
- Легко запутаться

**Лучше выбрать один вариант и придерживаться его.**

---

## Примеры команд

### Если используете свой репозиторий (GitHub):

```bash
# Обычный workflow
git add .
git commit -m "Обновление"
git push origin main  # Push в GitHub → Amvera автоматически задеплоит
```

### Если используете репозиторий Amvera:

```bash
# Workflow с репозиторием Amvera
git add .
git commit -m "Обновление"
git push amvera main  # Push в Amvera → автоматический деплой
```

## Частые вопросы

**Q: Можно ли переключиться с одного варианта на другой?**  
A: Да, в любой момент в настройках Amvera можно отключить один репозиторий и подключить другой.

**Q: Нужно ли удалять код из одного репозитория при переключении?**  
A: Нет, код может оставаться в обоих местах, но лучше выбрать один основной.

**Q: Что если у меня уже есть код в репозитории Amvera, но хочу перейти на GitHub?**  
A: Просто подключите GitHub репозиторий в настройках Amvera. Код из репозитория Amvera можно оставить или удалить.

---

## Итоговая рекомендация

**Для большинства случаев: используйте свой репозиторий (GitHub/GitLab/Bitbucket)**

Это стандартная практика и удобнее для работы. Amvera просто будет автоматически деплоить при каждом push.
